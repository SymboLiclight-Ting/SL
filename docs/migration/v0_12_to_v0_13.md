# Migration Guide: v0.12 to v0.13

v0.13 is a release-candidate hardening release. It is expected to require no source migration for valid v0.12 `.sl` applications.

## Expected User Action

- Keep existing `.sl` source files unchanged.
- Re-run `slc fmt --check`, `slc check`, `slc test`, `slc schema`, and `slc doctor`.
- For database-backed apps, run `slc doctor <file.sl> --db <path-or-url>` and `slc migrate plan <file.sl> --db <path-or-url>`.
- Review `docs/freeze-candidate.md` if your tooling depends on JSON output or CLI behavior.

## Compatibility Notes

- v0.13 keeps the v0.12 generated HTTP error envelope.
- v0.13 keeps v0.12 CLI exit code behavior.
- v0.13 keeps Postgres as an optional dependency through `symboliclight[postgres]`.
- v0.13 does not execute migrations automatically.

## If A Regression Appears

Treat any required v0.12 source change as a release candidate blocker. Add a minimal compatibility fixture and update `docs/compatibility.md` before accepting the change.
