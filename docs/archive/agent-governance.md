# Archived Agent-Governance Direction

Earlier SymbolicLight notes explored an agent-native governance language centered on:

```text
Draft<T> -> Checked<T> -> Approved<T> -> Authorized<Action>
```

That direction is archived for now because IntentSpec already covers much of the task contract, permission, and acceptance-test layer.

The current SymbolicLight mainline is a spec-native application language that compiles `.sl` apps to Python.

The original long-form discussion remains in `SymbolicLight Language.md`.
